# CF Datasets

Counterfactual and probe datasets used by both the CODI (`codi-work/`) and Huginn
(`huginn-work/`) experiments. All inference scripts read from this folder via
the project root.

This directory contains:

1. **Counterfactual SVAMP variants** — five JSON files, each a different
   regeneration of the original SVAMP problems with controlled perturbations.
   Used for testing whether internal representations track *operator* vs
   *magnitude* independently. Generators ship alongside as `generate_*.py`.
2. **Number-isolation probe sets** — two synthetic JSON files that fix one
   axis (numerals or operator) while varying the other, for PCA-based
   disentanglement testing.
3. **CoT faithfulness labels** — judgments of teacher (Llama-3.2-1B) chain-
   of-thought traces on SVAMP, produced by GPT-5 via `judge_cot.py`.

---

## 1. Counterfactual SVAMP variants

All five are derived from the same source — the 1000 ChilleD/SVAMP problems —
by extracting the equation, swapping in new numerals according to a control
scheme, regenerating the problem text, and (optionally) filtering. Each row
preserves the original SVAMP `idx` and operator type.

| File | Rows | Control | Key fields | Used by |
|---|---:|---|---|---|
| `cf_magmatched.json` | 972 | Numerals chosen so the *output magnitude bucket* is matched across operators (every operator appears in every <10 / 10–99 / 100–999 / 1000+ bucket evenly). Decouples operator from output magnitude. | `cf_question_concat`, `cf_answer`, `type`, `output_bucket` | early CODI Llama-1B `cf_lda_*` slideshows |
| `cf_balanced.json` | 676 | Iteratively rebalanced so **both input and output magnitudes** are simultaneously controlled per operator (within feasibility). The strictest of the four. | `cf_question_concat`, `cf_answer`, `type`, `input_bucket`, `output_bucket` | the canonical CF set used by *all* current `cf_lda_*` scripts (CODI + Huginn) |
| `cf_under99.json` | 642 | Numerals constrained to ≤ 99 (single- and double-digit only). Tests whether the operator code generalizes when magnitude is restricted to a small range. | `cf_question_concat`, `cf_answer`, `type`, `input_bucket`, `output_bucket` | sensitivity check; not in the headline analyses |
| `cf_under99_b.json` | 622 | Same as `cf_under99` but with a second iterative rebalancing pass — input and output buckets are both more uniform. | same as above | sensitivity check |
| `cf_gpt_transformed.json` | 546 | Each problem rewritten by GPT-5 to use new numerals AND new surface scenario/wording, while preserving the operator. The most lexically-diverse CF; closest to a "fresh" problem distribution. | `cf_question`, `cf_answer`, `type` | dual-label analysis in `visualizations-student-correct/sc-v3/` |

**Generators** (each fully reproducible, deterministic given the same seed):

| Script | Strategy |
|---|---|
| `generate_cf_magmatched.py` | Single-pass: pick numerals so each (operator, output_bucket) cell hits a target count. |
| `generate_cf_balanced.py` | Iterative-bucketing: alternates re-sampling input and output buckets until both marginals are balanced. |
| `generate_cf_under99.py` | Like `cf_magmatched` but with the additional `numerals ≤ 99` constraint. |
| `generate_cf_under99_b.py` | Like `cf_balanced` with the `≤ 99` constraint. |
| `generate_cf_gpt_transformed.py` | Calls GPT-5 with a strict prompt to rewrite each problem with new numerals + new wording while preserving the operator. |

### Schema (CF variants)

All five share the following per-row fields (some are absent in the
GPT-transformed set):

```
idx                  : SVAMP index 0..999  (preserves identity across CF sets)
src_idx              : same as idx (legacy field)
type                 : operator label, one of
                       {Subtraction, Addition, Multiplication, Common-Division}
orig_equation        : the original SVAMP equation, e.g. "( 12.0 - 5.0 )"
orig_body, orig_question : the original SVAMP body and question (if applicable)
cf_subs              : the new numerals as [a, b]
cf_body, cf_question : the rewritten body and question
cf_question_concat   : `cf_body + " " + cf_question` — what the model is shown
input_bucket         : "<10" | "10-99" | "100-999" | "1000+" — bucket of the
                       larger input numeral
output_bucket        : "<10" | "10-99" | "100-999" | "1000+" — bucket of cf_answer
cf_answer            : the gold numeric answer for the new numerals
```

`cf_gpt_transformed.json` does NOT include `cf_question_concat`, `input_bucket`,
or `output_bucket` — only `cf_question`, `type`, `cf_answer` and the source
metadata.

## 2. Number-isolation probe sets

Two synthetic datasets generated by `generate_number_isolation.py`, designed to
test whether number-encoding directions and operator-encoding directions live
in *orthogonal* subspaces of the residual stream.

| File | Rows | Held constant | Varies | Coloring axes for PCA |
|---|---:|---|---|---|
| `vary_numerals.json` | 80 | scenario template (always Subtraction: "John has *a* apples, gives *b* away…"), operator | the two numerals `(a, b)` with `5 ≤ b < a ≤ 200` | `a` (continuous), `b` (continuous), `answer` (= a−b, continuous) |
| `vary_operator.json` | 24 | the two numerals `(a, b) = (12, 4)` — same in every prompt | operator (4 ops × 6 scenarios per op) | `operator` (4-class discrete) |

PCA on `vary_numerals` activations isolates *number*-encoding directions (the
only thing that varies across the dataset). PCA on `vary_operator` isolates
*operator/scenario*-encoding directions (numerals contribute zero variance).
Comparing the principal subspaces of the two — `‖U_A^T U_B‖_F^2 / k` — gives
a direct measure of entanglement.

Used by:
- `huginn-work/visualizations/number_isolation_pca.py` — quantitative
  subspace-alignment heatmap.
- `huginn-work/visualizations/number_isolation_combined.py` and
  `codi-work/visualizations-all/gpt2/number_isolation_combined.py` — slideshow
  PDFs showing 2D + 3D PCA per (block/layer × step), colored by `a` / `b` /
  `answer` for vary_numerals and by operator for vary_operator.

### Schema (isolation sets)

```
idx               : 0-indexed
a, b              : input numerals (constant in vary_operator, varied in vary_numerals)
type              : operator label (constant in vary_numerals, varies in vary_operator)
answer            : ground-truth numeric answer for this (a, b, type)
question_concat   : the prompt text shown to the model
```

## 3. CoT faithfulness labels

| File | Source | Labels |
|---|---|---|
| `svamp_judged.json` | GPT-5 judging Llama-3.2-1B teacher's CoT on SVAMP | `faithful` / `unfaithful`. Only present for SVAMP problems where the teacher's *final answer* matches gold; the rest of SVAMP is implicitly `teacher_incorrect`. |
| `svamp_unfaithful.json` | filtered subset of `svamp_judged.json` | only the unfaithful examples (with reasoning text). |

Generator: `judge_cot.py` (calls GPT-5 with a structured-output prompt asking
for a binary faithful/unfaithful label plus the index of the first wrong step).
